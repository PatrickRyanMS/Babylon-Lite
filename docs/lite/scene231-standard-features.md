# Scene 231 ‚Äî Standard Material Features (skeleton / morph / vertex color / UV offset)

Reserved scene number **231** (Standard-material skeleton, morph targets, vertex color, UV offset).
This is **new Lite feature work** ‚Äî there is NO Babylon.js reference to diff against, so the scene

## Scene design (`lab/lite/src/lite/scene231.ts`)

1. `loadGltf(engine, "/test-assets/scene231-standard-features.glb")` ‚Üí meshes with `.skeleton`,
   `.morphTargets`, `_gpu.colorBuffer` (the glTF loader sets these material-agnostically; the shared
   `_computeMeshFeatures` reads them for any material family).
2. Call the Standard opt-ins so the (net-neutral) feature dispatch is installed:
   `enableStandardSkeleton()`, `enableStandardMorph()`, `enableStandardVertexColor()`,
   `enableStandardUvOffset()` ‚Äî these must be **exported from `index.ts`** (currently internal).
3. Replace each loaded mesh's material with `createStandardMaterial()` + a small checker/grid diffuse
   texture + `mat.uvOffset = [0.25, 0.1]` (so the offset is visible).
4. Fixed capture camera + a deterministic light; `?capture=1` for the golden render (mirrors scene230).

## RESOLVED ‚Äî vertex-color stride mismatch (fixed via float32x4 unification)

The glTF loader normalizes `COLOR_0` to a **tight float32x4 RGBA (stride 16)** buffer ‚Äî the engine-wide
vertex-color layout (PBR, procedural meshes, node/shader materials all bind `float32x4`). The
**Standard `stdVertexColorExt` originally ported for FBX declared `float32x3` (stride 12)** (FBX produces
tight RGB), so binding a stride-16 buffer to a stride-12 attribute misaligned every vertex ‚Üí garbage
colors.

**Resolved:** the Standard vcolor StdExt was unified to the engine-wide `float32x4` RGBA layout (reads
`color.rgb`, vertex alpha forced to 1.0, multiply-into-baseColor behavior unchanged ‚Äî Babylon.js parity),
and the FBX loader (Branch B) was updated to emit RGBA; m02 (FBX n-gons vcolor) re-confirmed unchanged at
MAD 0.041. scene231 now exercises **all four features** (skeleton + morph + vertex color + UV offset)
through the glb‚ÜíStandard path cleanly.
